# Header Utility Controls Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Convert the shared header theme toggle to localized text, keep language/theme controls fixed beside the brand, and prevent time-slot text selection during dashboard drag.

**Architecture:** Keep the existing Django base template and Bootstrap navbar. Add a small source CSS utility row for the language selector and theme toggle, update the inline theme script to manage text/accessibility state, and add scoped CSS to the dashboard time-grid interaction area.

**Tech Stack:** Django templates, gettext/i18n, Bootstrap 5 navbar, source CSS in `apps/core/static/core/css/style.css`, pytest.

---

## Approved Scope

- Modify `templates/base.html` header markup and theme toggle script.
- Modify `apps/core/static/core/css/style.css` for header utility layout and time-grid text-selection prevention.
- Update locale catalogs if new visible/aria strings are added.
- Add focused tests in existing core/dashboard test modules.
- Update post-work documentation and `docs/project-status.md` after verification.

## Non-Scope

- No full navbar redesign.
- No generated `staticfiles/` edits.
- No dashboard data model, API, save/delete, or selection behavior changes.
- No auth menu or navigation route changes.

**Files:**

**Step 1: Implement minimal template structure**

- Wrap the language form and theme button in a `.navbar-utility-controls` container immediately after `.navbar-brand`.
- Add class names for the language form/select so inline styles can move to CSS.
- Replace the theme button icon with a text span, initially using `{% trans "다크" %}`.
- Add data attributes for localized labels:

  - `data-label-light`
